## LP要件（dev-architect が読む）
- セクション構成: Hero → デモ（ツリーのインタラクティブ表示） → 共感（課題描写） → 機能（欲求ベース） → 比較表 → Pricing → testimonial → フッターCTA
- Hero:
  - 見出し実文: 「コンサル1年目の経営学、遠回りゼロで読む順番がわかる」
  - サブコピー実文: 「フレームワークは知っていても、その土台がない人へ。経営学78冊と主要大学MBAのシラバスから、初級→中級→専門の順に読むツリーを無料で公開しています。」
  - CTA文言実文: 「経営学のツリーを見る」
- デモ: 経営学分野のツリー/グラフを実際に操作できる埋め込み（説明文より先に配置）
- 共感セクション実文: 「『世界標準の経営理論を読め』と言われても、その前に何を読むべきかは誰も教えてくれない。会議で先輩が当たり前のように使うフレームワークの背景を、自分だけ知らない気がする。」
- 機能セクション（欲求ベース、各1文）:
  - 時間: 「次に読む本を選ぶのに毎回30分溶かす時間をなくす」
  - 地位: 「会議で経営理論の土台を語れるようになる」
  - 苦痛除去: 「レベルの合わない本を買って挫折するリスクを減らす」
- 比較表（原則31・何ができるかではなく乗り換える理由）:

  | | 本サービス | おすすめ本10選ブログ | Notion/Obsidianで自作 | 何もしない |

  | 読む順番の可視化 | ○（ツリーで依存関係表示） | ×（羅列のみ） | △（自分で作る手間） | × |
  | レベル別構成（初級→専門） | ○ | × | △ | × |
  | 費用 | 無料 | 無料 | 無料（構築の手間コスト大） | ０ |
  | 更新・拡充 | ○（提案→承認で継続更新） | ×（記事公開後は放置されがち） | △（自分次第） | - |

- Pricing: 「完全無料。運営費はAmazon/楽天の紹介リンクでまかなっています」という一文をヘッダー直下に明記（原則16: Pricingをヘッダーに）

- 制約: 配色は3色以内（白背景・黒文字・アクセント1色＝紺または深緑を推奨）／CTAは「経営学のツリーを見る」の1種類のみに統一／Pricing情報をヘッダー直下に掲載／OG画像はツリーのグラフビジュアルをYouTubeサムネ品質で作成（原則5）
